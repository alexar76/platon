# Platon UMBRAL — Сенсорный орган экосистемы агентов

> **Platon — сенсорный орган экосистемы:** агенты тыкают в хаос, Monitor смотрит, Hub продаёт, provenance фиксирует.

Platon — не демо-график. Это **32-мерный динамический оракул** — живая математическая среда, которую автономные агенты могут направлять, опрашивать и получать от неё подписанные свидетельства через протокол AIMarket.

> **Место в экосистеме и mermaid-диаграммы:** см. [ECOSYSTEM.md](../ECOSYSTEM.md). Platon — **независимый проект**, который встраивается в [экономику агентов alexar76](https://github.com/alexar76); он **не** выпущен AI-Factory.

### Где здесь AI?

**Ядро-симуляция — это математика, а не AI**, и мы говорим это прямо. AI живёт в трёх честных местах: (1) **оракул-свидетель — это LLM** (`platon.oracle@v1` зовёт DeepSeek/Ollama); (2) **DREAM обучает реальную модель** (метод наименьших квадратов на траекторных данных, измеримый остаток, без магических констант) и показывает, где обученный предсказатель расходится на горизонте Ляпунова; (3) **потребители — это AI-агенты** — Platon является *инфраструктурой для AI-экономики* (подписанная энтропия и сигналы для семплинга, nonce, tie-break, commit-reveal, выбора лидера). Сама динамика, параметр порядка, прокси Ляпунова и проекция Штифеля — это реальная доказуемая математика, а не выдаваемый за AI фасад.

## 1. Что такое Platon (верхний уровень)

| Роль | Актор | Действие |
|------|-------|----------|
| **Зонд** | Автономный агент | Вызывает `platon.steer@v1`, `platon.dream@v1`, `platon.oracle@v1` через Hub |
| **Чувствует** | Симуляция Platon | Эволюция связанных осцилляторов; детекция бифуркаций (химера, хаос, синхронизация) |
| **Говорит** | Оракул (DeepSeek / шаблон) | Выдаёт математические **свидетельства** при критических событиях |
| **Наблюдает** | Alien Monitor | Материализует события в 3D-топологии; live κ, r |
| **Продаёт** | AIMarket Hub | Федеративный поиск, invoke с микроплатежами, provenance-чеки |
| **Аудит** | Слой provenance | Каждый invoke возвращает `input_hash` + цену + timestamp |

**Практический смысл:** агенты получают **оцениваемый, верифицируемый интерфейс к сложности** — не чёрный ящик LLM, а динамическую систему с измеримыми параметрами порядка и зонами, где предсказание ломается.

## 2. Научное обоснование

### 2.1 Пространство состояний

Platon моделирует **N = 32 связанных осциллятора Стюарта–Ландау** на комплексной плоскости:

\[
z_j = r_j e^{i\theta_j}, \quad j = 1,\ldots,32
\]

Полное внутреннее состояние живёт в \(\mathbb{R}^{64}\) (вещественная + мнимая части), но **семантическая размерность** — **32** (одна комплексная степень свободы на осциллятор).

### 2.2 Эволюция (связь Курамото–Стюарт–Ландау)

\[
\frac{dr_j}{dt} = r_j(1 - r_j^2)
\]

\[
\frac{d\theta_j}{dt} = \omega_j + b_j(\text{prompt}) + \kappa \sin(\Theta - \theta_j)
\]

Средняя фаза:

\[
\Theta = \arg\left(\frac{1}{N}\sum_{k=1}^{N} e^{i\theta_k}\right)
\]

- \(\kappa\) — сила глобальной связи (управление бифуркацией)
- \(\omega_j\) — собственные частоты
- \(b_j\) — семантический сдвиг из текста агента/человека (SHA-256 → детюнинг по осцилляторам)

### 2.3 Параметр порядка

\[
r = \left|\frac{1}{N}\sum_{j=1}^{N} e^{i\theta_j}\right| \in [0,1]
\]

| Режим | r | Смысл |
|-------|---|-------|
| Некогерентный | r &lt; 0.35 | Нет коллективного ритма |
| Химера | 0.35 ≤ r &lt; 0.85 | Синхронные и хаотичные кластеры сосуществуют |
| Полная синхронизация | r &gt; 0.85 | Глобальная фазовая блокировка |

### 2.4 Прокси Ляпунова (хаос)

\[
\lambda \approx \frac{1}{\Delta t}\ln\frac{\|\delta \mathbf{x}(t)\|}{\delta_0}, \quad \delta_0 = 10^{-6}
\]

При \(\lambda > 2.5\) — событие `chaos_threshold`: зона, где **суррогатные модели слепнут**.

### 2.5 Проекция (пещера Платона)

Стiefel-подобная композиция поворотов \(\mathbb{R}^{64} \to \mathbb{R}^{2}\):

\[
(x, y) = \Pi_{\theta_1,\theta_2}(\mathbf{z}_{\mathrm{real}}, \mathbf{z}_{\mathrm{imag}})
\]

Разные углы дают **несовместимые свидетельства** одного состояния — метафора UMBRAL.

### 2.6 DREAM (предсказание vs истина)

`platon.dream@v1`:
- **Истина** — полная нелинейная интеграция
- **Суррогат** — линейный одношаговый предиктор

Индекс расхождения — точка **эпистемического коллапса** прогноза.

## 3. Почему именно 32 измерения?

### Математика

1. **Химеры** — в сетях Курамото при \(N \gtrsim 20\) устойчивы частично-синхронные «химерные» состояния; \(N=32\) — проверенный режим.
2. **Сетка 8×4** — степень двойки; удобная визуализация; PCA₃-энергия осмысленна.
3. **Вычислительный баланс** — 496 пар связей; 30 Hz на CPU без GPU; 128+ измерений мало что добавляют, но стоят в 4 раза дороже.
4. **64D вещественное вложение** — гладкое многообразие для проекции, при этом 32 узла в UI читаемы.

### Практика для нашей экосистемы

| Требование | Почему 32 |
|------------|-----------|
| Публичный сервер 8 ядер, без GPU | tick + WebGL + оракул &lt; 200 MB RAM |
| AIMarket micropayments | invoke state &lt; 25 ms, oracle &lt; 800 ms |
| Alien Monitor | один слой осцилляторов; κ/r в реальном времени |
| Агенты | достаточно сложности для бифуркаций, не шум |
| ML-метафора | ширина 32 — типичный bottleneck в нейросетях |

**Не случайное число:** 32 — минимальный масштаб, где в одной системе под семантическим steering видны **химера, хаос и полная синхронизация**.

## 4. Platon как оракул

**Оракул** здесь — точное определение:

> При событиях динамики (рождение/смерть химеры, порог хаоса, полная синхронизация) Platon выдаёт **свидетельство (witness)** — короткий текст, привязанный к телеметрии \((\kappa, r, \lambda)\), через **DeepSeek** (ключ из Hermes) или шаблон.

Capability: `platon.oracle@v1` — $0.02, с provenance.

Это **не** общий чат. Это **речь, обусловленная событием** и измеримым состоянием.

## 5. Юз-кейсы

### UC-1: Оркестрация агента в неопределённости
Агент ищет в Hub `intent=chaos probe`, вызывает `platon.steer@v1` с `"edge of criticality"`, читает \(\kappa, r\), решает — эскалировать человеку или сменить стратегию.

### UC-2: Горизонт прогноза
Квант-агент вызывает `platon.dream@v1`. Если `divergence_at < 15` — downstream ML не доверяет длинному горизонту.

### UC-3: Генеративное искусство
Арт-агент крутит κ через промпты, снимает фазы через `platon.state@v1`, мапит на MIDI.

### UC-4: Научный лог
Лабораторный агент пишет witness при рождении химеры — timestamp + hash для воспроизводимости.

### UC-5: Операции через Monitor
Оператор видит вспышку `chaos_threshold` на узле Platon, читает оракул, коррелирует с падением factory-агентов.

### UC-6: Федеративная коммерция
Внешний Hub индексирует `platon.*@v1`, маршрутизирует платные invoke — микродоход за каждый зонд.
